== Project goals

This project aims to provide a *clean, explicit, and testable* implementation of a Quacks-style board game engine.

Key objectives:

* Model the full game loop (rounds, phases, events) in a structured way
* Centralize and make game state explicit and auditable
* Replace hard-coded branching logic with dispatch- and table-driven rules
* Enable incremental implementation of event cards, chips, and expansions
* Provide developer-friendly console output for debugging and verification

Non-goals (for now):

* Graphical user interface
* Online or networked multiplayer
* Complete coverage of all expansions and edge rules

Correctness, clarity, and maintainability take precedence over feature completeness.
